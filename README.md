# 🎯 InterviewAI — Interview Platform

A fully client-side AI-powered interview platform built in a single HTML file. No backend, no server — just open in a browser and go.

---

## ✨ Features

### 👥 Two Roles

| Role | What They Can Do |
|------|-----------------|
| **Admin** | Post interview listings, manage & delete them, view all applicants and scores |
| **Applicant** | Browse posted interviews, apply, take the AI interview, view detailed report |

### 🛡️ Admin Panel
- **Post Interview** — create a listing with job title, company, interview type, difficulty, question count, time limit, and job description
- **Manage Interviews** — view all your postings, see how many people applied, delete listings
- **Dashboard** — see total interviews posted, total applications, and average score across all applicants
- **View Applicants** — click any listing to see a table of who applied and their scores

### 👤 Applicant Panel
- **Browse Interviews** — see all available interview listings from admins
- **Apply & Interview** — start an AI-powered interview session for any listing
- **My Applications** — track all your past interviews and results

### 🤖 AI Interview Engine
- Questions generated by **OpenAI ** tailored to the job description, role, type, and difficulty
- Falls back to a built-in question bank if the API is unavailable
- Interview types: Technical, Behavioral, HR/General, Leadership, Product/PM, Mixed
- Difficulty levels: Junior, Mid, Senior, Lead
- Optional per-question time limit (countdown timer)
- **All feedback is generated at the end** — no interruptions during the interview
- Batch evaluation: Claude scores and provides feedback for all answers at once

### 📊 Report
- Overall score with animated ring chart
- Per-question score badges (Excellent / Good / Fair / Needs Work)
- Expandable breakdown: your answer + AI feedback for each question
- Download report as a `.txt` file
- Admins can see applicant scores in their dashboard

## 🚀 Getting Started

### 1. Open the file

Just open `index.html` in any modern browser (Chrome, Edge, or Firefox).

```
No installation. No npm. No server needed.
```

### 2. Create an account

- Choose your role: **Admin** or **Applicant**
- Sign up with any email and password (stored locally in your browser)
- Three demo interview listings are pre-loaded for applicants to browse

### 3. Admin: Post an interview

1. Log in as **Admin**
2. Go to **Post Interview**
3. Fill in the job details and click **Post Interview**
4. Your listing appears under **Manage Interviews** and in the applicant's **Browse** tab

### 4. Applicant: Take an interview

1. Log in as **Applicant**
2. Go to **Browse Interviews**
3. Click **Apply & Interview** on any listing
4. Answer each question in the text box and press Enter or click ➤
5. After the last question, AI evaluates all answers and shows your full report

##🔑 OpenAI API
The app calls the OpenAI API directly from the browser to:

Generate targeted interview questions

Score and provide feedback on all answers (batch, at the end)

To enable AI features, the app needs to reach 'https://api.openai.com.' If you're running this locally or on a server that proxies the API, it will work out of the box.
​

⚠️ Note: Calling the OpenAI API directly from a browser exposes your API key if you hard‑code it. For production use, route calls through a backend proxy.
​

If the API is unavailable, the app gracefully falls back to a built‑in question bank and provides basic scoring.

## 🔐 Auth & Data

- User accounts are stored in `localStorage` under the key `iai_users`
- Passwords are hashed (djb2 algorithm) before storing
- Sessions persist in `localStorage` across page refreshes
- All job listings and applications are stored in `localStorage`
- **Data is local to the browser** — clearing browser data will reset everything

---

## 🧩 Interview Flow

```
Applicant applies → Questions generated (Claude API or fallback bank)
       ↓
Q1 displayed → Applicant types answer → Submit
       ↓
Q2 displayed → ... → Last question answered
       ↓
All answers sent to Claude in one batch → Scores + feedback returned
       ↓
Full report shown (scores, feedback per question, overall grade)
```
