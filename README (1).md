# 🎙️ VoiceEquity — AI-Powered Voice Interview Platform

> *Breaking barriers in hiring through voice-first AI technology.*

---

## 🌍 What is VoiceEquity?

VoiceEquity is a fully client-side, AI-powered interview platform built for **accessibility and fairness**. It lets candidates complete job interviews using only their voice — no reading, no typing, no visual dependency required. Admins create and manage interview templates; candidates speak their answers and receive instant AI-generated feedback and scores.

Everything runs in the browser. No backend. No installation. Just open and go.

---

## ✨ Key Features

### 👥 Two Roles

| Role | Capabilities |
|------|-------------|
| **Admin** | Create interview templates, set difficulty and type, manage listings, view all candidate scores and results |
| **Candidate** | Browse available interviews, complete voice sessions, receive AI feedback, track past results |

### 🎤 Voice-First Interface
- Speak your answers — no typing required
- Live audio waveform and transcript display
- Real-time microphone toggle with ripple animation
- Repeat last question or skip with single button press
- Fully keyboard navigable and screen reader compatible

### 🤖 AI Interview Engine (Claude-Powered)
- Questions generated by **Claude (claude-sonnet-4)**, tailored to the job description, role, type, and difficulty level
- Built-in fallback question bank if API is unavailable
- Batch evaluation — all answers scored together at the end, no interruptions during the session
- Per-question score (0–100) with detailed written feedback

### 📊 Detailed Score Reports
- Animated score ring showing overall performance
- Grade label: Excellent / Strong / Good / Needs Work / Keep Practising
- Per-question breakdown with your answer and AI feedback
- Downloadable `.txt` report
- Admin dashboard shows all candidate scores with timestamps

### 🛡️ Auth & Data
- Sign up / sign in with email and password (hashed, stored locally)
- Guest mode for instant access without registration
- Admin protected by a separate access key
- All data stored in `localStorage` — fully private, no server required

## 🚀 Getting Started

### 1. Open the App
Just open `index.html` in any modern browser. No npm, no server, no install.

### 2. Choose Your Role
- **Candidate** — sign up or continue as guest, then browse and take interviews
- **Admin** — sign in with admin credentials and access key to manage templates

### 3. Admin: Create an Interview
1. Log in as Admin
2. Go to **Interviews** panel
3. Click **+ Create Interview**
4. Set title, role, type, difficulty, question count, and optional job description
5. Save — it's instantly live for candidates

### 4. Candidate: Take an Interview
1. Log in or continue as Guest
2. Browse available interviews
3. Click **Start Voice Interview**
4. Speak your answers when prompted
5. Receive your full AI-scored report at the end

## 💡 Account Innovation

VoiceEquity rethinks what "account" means in a hiring tool:

- **No friction onboarding** — candidates can join as a guest with one click, removing the barrier of mandatory registration for first-time users
- **Role-based access** — a single auth flow supports both candidates and admins, with the experience adapting automatically to the user's role
- **Admin key protection** — admin accounts require a separate access key on top of email/password, preventing unauthorized escalation of privileges
- **Persistent sessions** — once signed in, users return directly to their dashboard without re-authenticating, making the experience seamless across revisits
- **Privacy by default** — all account data lives only in the user's browser. There is no central server, no data lake, and no risk of a breach exposing candidate information

### What Makes It Different

**Voice-first is not an afterthought.** Most AI interview platforms add voice as an optional feature on top of a text-based system. VoiceEquity is designed from the ground up so that voice is the *only* interaction method — making it genuinely usable for candidates with visual impairments, dyslexia, or limited literacy.

**No interruptions.** Unlike tools that score and comment after every answer, VoiceEquity waits until the full interview is done before sending anything to the AI. This mirrors a real interview — you finish, then get feedback — rather than a gamified quiz.

**Context-aware questions.** When an admin pastes a job description, Claude reads it and generates questions specific to that role, seniority level, and interview type. This is not a random draw from a static bank — it is a dynamic, context-aware question set every time.

**Equity in the name, equity in the design.** The platform is aligned with SDG 4 (Quality Education), SDG 9 (Innovation), and SDG 10 (Reduced Inequalities). It is free, serverless, and built so that anyone — regardless of visual ability or economic access — can practice for a real job interview with AI-quality feedback.

### 🎤 Web Speech API — Browser-Native Open Standard

The entire voice layer of VoiceEquity — microphone capture, real-time transcription, and text-to-speech question delivery — is powered by the **Web Speech API**, a W3C open web standard built into modern browsers. No third-party library or paid speech service is required.

| Feature | API Used | Spec |
|---------|---------|------|
| Speech-to-text (candidate answers) | `SpeechRecognition` / `webkitSpeechRecognition` | [W3C Web Speech API](https://wicg.github.io/speech-api/) |
| Text-to-speech (question playback) | `speechSynthesis` | [W3C Web Speech API](https://wicg.github.io/speech-api/) |
| Audio waveform visualisation | HTML5 `<canvas>` + `CanvasRenderingContext2D` | [WHATWG HTML Living Standard](https://html.spec.whatwg.org/multipage/canvas.html) |

**Reference:** WICG Web Speech API specification — [wicg.github.io/speech-api](https://wicg.github.io/speech-api/)

---

### 💾 Web Storage API — Browser-Native Open Standard

All user accounts, interview templates, and results are stored using **Web Storage**, a W3C open standard available natively in every modern browser. No database, no server, no third-party storage service.

| Storage Mechanism | Used For | Spec |
|------------------|---------|------|
| `localStorage` | User accounts, interview templates, results | [WHATWG Storage Living Standard](https://storage.spec.whatwg.org/) |
| `sessionStorage` | Active interview state, report data between pages | [WHATWG Storage Living Standard](https://storage.spec.whatwg.org/) |

**Reference:** WHATWG Storage specification — [storage.spec.whatwg.org](https://storage.spec.whatwg.org/)

---

### 🔐 djb2 Hash Algorithm — Public Domain

Passwords are hashed client-side using the **djb2** algorithm, a lightweight non-cryptographic hash function created by Daniel J. Bernstein and released into the public domain. It is used here for basic obfuscation of stored passwords in localStorage.

> ⚠️ Note: djb2 is not a cryptographically secure hashing algorithm. For production deployments, replace with a server-side solution using bcrypt or Argon2.

**Reference:** [Daniel J. Bernstein — djb2](http://www.cse.yorku.ca/~oz/hash.html) (public domain)

### 🤖 Anthropic Claude API — AI Question Generation & Scoring

The AI engine calls the **Anthropic Messages API** to generate tailored interview questions and score candidate answers. The model used is `claude-sonnet-4-20250514`.

| Usage | Endpoint | Model |
|-------|---------|-------|
| Generate interview questions from JD | `POST https://api.anthropic.com/v1/messages` | `claude-sonnet-4-20250514` |
| Batch score and give feedback on all answers | `POST https://api.anthropic.com/v1/messages` | `claude-sonnet-4-20250514` |

**Reference:** Anthropic API Documentation — [docs.anthropic.com](https://docs.anthropic.com)

> ⚠️ Calling the Anthropic API directly from the browser exposes your API key if hard-coded. For production, route API calls through a backend proxy.

<a role="button" tabindex="0" aria-label="Enter as Candidate">…</a>
